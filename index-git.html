Diccionario Programación: GIT

GIT es un programa de desarrollo abierto que permite controlar las versiones de control de los proyectos.

Instalación

Para instalara GIT en tu computadora puedes acceder al sitio oficial de GIT y escoje la opción de descarga según tu sistema operativo. para windows se descarga GIT Bash que es la consola con la que se trabaja a continuación.

Configuración de la terminal

configuracion global GIT

permite definir un nombre de usuario y correro con "git config --global usermane "nombre"

Abrir carpeta proyecto

al abrir la carpeta que contiene el proyecto, es posible si git esta instalado abrir directamente en esa ubicación, para ello dando click derecho se busca la opcion "git Bash here", esto abre la carpeta en la terminal Bash

Inicializar repositorio

Para inicializar el repositorio de nuestro archivo, solo es necesario usar el comando "git init"

$ git init


Validar archivos en GitBash

para verificar que archivos estan cargados en GIT, en la terminal se utiliza el comando "git status -s" que como resultado da un listado con todos los archivos, inicialmente todos los archivos van con un par de interrogaciones (??) en color rojo indicando que los archivos son visibles para GIT pero no les esta realizando ningun seguimiento o acción.

$ git status -s


Seguimiento a un archivo particular

Si se buscar hacer seguimiento en Git de un archivo en especifico ubicados en una carpeta se utiliza el comando "git add archivo.extencion", en el listado de estatus el archivo aparace con una "A" en color verde o area de ensayo.

$ git add index.html


Seguimiento a todos los archivos

Si se buscar hacer seguimiento en Git a todos los archivo de un proyecto especifico se utiliza el comando "git add ." donde archivo.extencion es el nombre del archivo y su correspondiente extensión, en el listado de estatus todos los archivos aparacen con una "A" en color verde o area de ensayo.

$ git add .


Enviar al repositorio

Para enviar todos los archivos que se encuentren en seguimiento al repositorio se usa el comando "git commit -m "descripción", ya estos archivos no son visibles en el listado de estado

$ git commit -m "descripción"


Guardar cambios en un archivo

Si posterior al envio al repositorio, un archivo es modificado y se guarda los cambios, este en la lista de estado aparece de nuevo con una letra "M" en color rojo indicando que el archivo fue modificado, posteriormente se debe enviar a la zona de seguimiento con el comando "git add archivo.extencion" quedando en la lista de estado con la "M" en verde y finalmente se envia al repositorio con el comando "git commit -m "descripción"

$ git add index.html
$ git commit -m "Se agrega mas contenido a la página"


Enviar al repositorio y guardar cambios

Para enviar y guardar todos los archivos que se encuentren en seguimiento al repositorio se usa el comando "git commit -am "descripción 2", ya estos archivos no son visibles en el listado de estado

$ git commit -am "Se agregan otros archivos"


Revisar repositorios

Para revisar cuantos repositorios o versiones se han guardado con exito se utiliza el comado "git log --oneline" desplegando el listado con sus correspondientes comentarios

$ git log --oneline"


Restaurar un archivo a una version anterior

para restaurar un archivo a una versión previa guardada en un repositorio es necesario utilizar el comando "git reset -hard codigo" donde codigo es una combinacion alfanumerica unica para cada archivo asignado en la lista de estado

$ git reset --hard 76a6140"


Enviar repositorio a GitHub

Para poder enviar el repositorio local de GIT a GitHub primero se debe establecer comunicacion entre los dos destinos con el comando git "push origin master", luego se envia el repositorio a GitHub con el comando "git remote add origin https://github.com/usuariogit/nombre-repositirio.git", esta direccion es suministada por GitHub en su panel. para VSC existe un atajo, primero se crea un respositorio vacio en GitHub y se obtiene la dirrecion que suminstra de forma automatica, luego en VSC en la barra de herramientas se seleciona ver -> paleta de comados y en la paleta de comandos se utiliza el comando "git clone" luego nos solicita la URL del repositorio GitHub y luego muestra un explorador donde selecionaremos la carpeta del proyecto, finalmente en el panel git se selecciona ... -> push.

git push origin master
git remote add origin https://github.com/jpuerta523/Manual-programacion.git

Modificar archivo desde Github

GitHub permite modificar un archivo directamente desde el repositirio web, simplemente se accede al archivo y se modifica desde el icono con forma de lapiz, finalmente se guardan con el comado "git pull" los cambios en el boton verde del fondo y automaticamente guarda los cambios tanto en el repositorio como en el equipo.

git pull

Manejo de versiones

es posible en git poner Tags (etiquetas) para dar numero de version al repositorio, para ello en la terminal GitBash se pone el codigo "git tag info de version -m "descripción", para caragar los cambios en GitHub usamos el comando "git push --tags"

git tag -m 21/11/2022v1 -m "calculadora version freecodcamp"
git push --tags

Clonar un repositorio

Desde un repositorio en Github se busca el icono verde "code" y se copia la dirreción web que ofrece,luego abrimos una terminal GitBash enlazada a la carpeta local de destino y utilizamos el comado "git clone direccion-web-copiada"

git clone https://github.com/jpuerta523/calculadora.git

Ramas o branches

Las ramas son versiones en paralelo que permiten trabajar en el proyecto en paralelo al hacer cambios sin que afecten otras versiones para posteriormente se fusionadas en una sola rama, generalmente en la rama principal conocida como master


Crear una nueva rama

Para crear una nueva rama del proyecto solo es neceario usar el comando "git branch nombreRama", con esto ya queda creada la nueva rama con el nombre asigando

git branch rama1

fusionar ramas

En la rama que se actualizara, que generalmente es la master se utiliza el codigo "git merge nombreRama" donde nombreRama es la rama a fusionar con la principal. si hay modificacines en el mismo archivo en las dos ramas el GitBash informa de conflictos que deben ser solucionados en el editor de texto, VSC nos ofrece un atajo en el campo git -> ... -> merge branch -> selecionar rama

git merge rama1

Cambiar de rama

Primero es posible visualizar la rama que esta activa, para ello se usa el codigo "git branch" que muestra todas las ramas diponibles en el proyecto y en verde con un asterisco señala la rama activa. para activar otra rama se usa el codigo "git checkout nombreRama". en VSC existe un atajo para cambiar de rama accediendo al primer elemento a la izquierda de la barra inferior, al hacer click alli se desplega la lista de ramas disponibles y simplemente se da click en la requerida.

git branch
git checkout rama1

Elininar rama

Es posible eliminar una rama ya sera por que se aceptaron los cambios en la rama principal con merge o por que sus modificaciones no seran tenidas en cuenta para el proyecto, para ello se utiliza el comando "git branch -d nombreRama".

git branch -d rama1

Resolver conflictos entre ramas

Al juntar (merge) dos ramas es posible que conflictos debidos a que para una misma parte del codigo ambas ramas presente versiones diferentes, en este caso VSC al hacer merge nos informa del conflicto y nos opcion de dejar el codigo del master, el codigo de la rama o dejar ambos, adicional permite comparar ambos codigos en paralelo antes de tomar la desición, si se aceptan ambos cambios el documento en el panel de git aparece con una C y la rama master aparece con una "!" al final indicando que tiene conflictos, para resolverlo en donde sale el archivo en el panel de git aparece un "+" que permite enviar el archivo a stage area quedando como modificado con la letra "M" y disponible para hacer un commit
