Instalar Bootstrap: para una aplicacion sencilla sin ningun tipo de personalización u optimización es suficiente con tener a bootstrap como un enlace via CDN copiando el link: link href="https://cdn.jsdelivr.net/npm/bootstrap@5.2.3/dist/css/bootstrap.min.css" rel="stylesheet" integrity="sha384-rbsA2VBKQhggwzxH7pPCaAqO46MgnOM80zW1RWuH61DGLwZJEdK2Kadq2F9CUG65" crossorigin="anonymous" en el head del documento HTML.
Si en cambio se necesita apliaciones mas complejas de bootstrap es necesario instalarlo fisicamente en el proyecto pra ello en la terminal de VSC en la carpeta del proyecto se ejecuta: "npm init -y" que genera package.json en el proyecto, luego se ejecuta en la terminal: "npm install bootstrap@5.2.3".
luego en el proyecto se crea una carpeta nueva llamada "sass" y en su interior un archivo llamado "custom.scss" y en su interior se pega el import suministrado en la pagina de bootstarp: "@import "../node_modules/bootstrap/scss/bootstrap";", finalmente se compila con el live Sass compiler ubicado en la parte inferior del VSC creando un archivo "custom.css" que contiene todas las clases diponibles de bootstrap. este custom.css es el archivo de estilo que se debe llamar en el head del documento HTML


Para modificar o agregar propiedades de las clases de bootstarp, esta se debe buscar en el archivo "custom.css" y copiarla para llevarla al archivo "custom.scss" aqui ya se cambia el valor de la clase por lo requerido en el css, ejemplo:
el el archivo custom.scss:
$primary: #555;
@import "../node_modules/bootstrap/scss/bootstrap";

depurar CSS: para generar un archivo de CSS mas liviano que contenga unicamente las clases de bootstrap utilizadas se utiliza purgecsspurgecss el comando: "npm i -g purgecss" luego se sige el proceso del video https://youtu.be/1kNwZbRiVcQ?list=PLPl81lqbj-4IcaAluUlCTmbYz0h9XQ8U1&t=4614

breakpoints:define los puntos donde cambia la distribución de la pnatalla para hacer contenido responsive con clases sm para menos de 576px hasta xxl para mas de 1400px.


container: todo el contenido del sitio es recomnedable que se ubique al interior de una etiqueta div con class "container", lo cual permitira centrar los contenidos, aplicar la grid y formatos de forma homogenea. puede llevar breakpoints para que ocupe el 100% hasta cierta medida

Grid: El espacio de trabajo cunado se activa el grid en bootstrap se divide en 12 columnas, para activar grid el contenido debe estar al interior de un div con clase "container" y los elementos que van en la misma fila van al interior de otro div con clase de row, para indicar cuantos espacios toma cada elemento lleva la clase "col" si solo toma una columna o "col-#" donde # indica la cantidad de columnas que toma, ejemplo: ...

Grid responsiva: para generar una grilla dinamica donde cada columna cambie de ancho que ocupa en la pantalla se define varias clases de "col-#-tamaño" en cada elemento empenzado siempre por el tamaño mas pequeño, ejemplo: class="col-12 col-sm-6 col-md-4" https://youtu.be/1kNwZbRiVcQ?list=PLPl81lqbj-4IcaAluUlCTmbYz0h9XQ8U1&t=2200

Flex:Permite utilizar las caracteristicas del flexbox en bootstrap como centrado o ubicacion de los elementos en algun extremo, para activarlo se da la clase "d-flex" en el elemento contenedor y a cada elemento la clase especifica segun la necesidad, para centrar en vertical se debe asignar un tamaño al elemento contenedor

offset, helpers: ratio, etiqueta font-family en el body, colores definidos como varibles en :root(https://youtu.be/ng8ur8KNrpU?list=PLvq-jIkSeTUbf3LXzO2QQa3P-oRMcrTAp&t=12043), favicon, api whatsapp https://youtu.be/ng8ur8KNrpU?list=PLvq-jIkSeTUbf3LXzO2QQa3P-oRMcrTAp&t=15729, order, hero image dinamico https://youtu.be/ng8ur8KNrpU?list=PLvq-jIkSeTUbf3LXzO2QQa3P-oRMcrTAp&t=20387, ventana modal

SASS

definir variables: primero se define la regla enmpezando "$"antes del nombre, ejemplo:

$main-fonts: Arial, sans-serif;
luego en las reglas:
h1 {
font-family: $main-fonts;
}

anidar reglas CSS: se pueden anidar las reglas CSS como en una estructura HTML

h1 { text-align: center;
color: blue;
}
p {
font-size: 20px;
}

mixin:

@mixin border-radius($radius){
-webkit-border-radius:$radius;
-moz-border-radius: $radius;
-ms-border-radius: $radius;
border-radius: $radius;

}

#awesome {
width: 150px;
height: 150px;
background-color: green;
@include border-radius(15px);

}

logica if en SASS

@mixin border-stroke($val) {
@if $val == light {
border: 1px solid black;
}
@else if $val == medium {
border: 3px solid black;
}
@else if $val == heavy {
border: 6px solid black;
}
@else {
border: none;
}
}

#box {
width: 150px;
height: 150px;
background-color: red;
@include border-stroke(medium);
}

importar archivos SASS

los archivos parciales se deben llamar con un "_"al inicio del nombre y con extencions SCSS, ejemplo: _variables.scss
para llamar el archivo se copia @import "variables" en el archibo destino

Extend: lleva todas las reglas CSS de un elemento y permite adicionarlas a otro elemento:

.info{
width: 200px;
border: 1px solid black;
margin: 0 auto;
}

.info-important{
@extend .info;
background-color: magenta;
}
